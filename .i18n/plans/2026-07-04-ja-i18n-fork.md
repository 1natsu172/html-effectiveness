# ja_JP i18n Fork + 週1自動追従 実装プラン

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 英語HTMLサンプル集33ファイルを日本語訳して `docs/` に置き GitHub Pagesで日本語公開し、週1でupstreamに自動追従して差分を再翻訳・自動pushする体制を構築する。

**Architecture:** `main`単一ブランチ。ルートの英語HTMLは無改変（翻訳の元ネタ兼マージ先）、日本語訳は `docs/` に構造ミラーで配置し GitHub Pages `/docs` ネイティブ配信。追従はGitHub Actionsではなく Claudeクラウド定期ルーティンで行い、`main`へ直接commit&push する。

**Tech Stack:** 静的HTML（ビルドなし）、GitHub Pages（Deploy from a branch: main /docs）、Claude `Workflow`（初回並列翻訳）、Claudeクラウドルーティン（週1追従）、`gh` CLI、`git`。

## Global Constraints

以下はspec由来の全タスク共通の不変制約。各タスクの要件に暗黙的に含まれる。

- **Fork衛生（最重要）**: upstream追跡ファイル（ルートの `*.html`, `README.md`, `LICENSE` 等）は**絶対に編集・削除しない**。Fork独自の追加物は `docs/` と `.i18n/` の**新規パスのみ**に置く。
- **翻訳する対象**: 画面表示テキスト / `<title>` / `<meta name="description">` / `alt`・`aria-label` / `<html lang="en">`→`<html lang="ja">`。
- **原文維持**: `<code>`/`<pre>`/`<script>`/`<style>`内、コード識別子・変数/関数/ファイル名、技術用語の英語表記、架空ブランド「Acme」等の固有名、先頭の `<!-- Copyright ... -->` ヘッダ。
- **文体**: 敬体（です・ます）。UIラベル・ボタン・短い見出しは体言止め可。
- **訳語統一**: `.i18n/glossary.md` を唯一の情報源として初回翻訳・週1ルーティンの両方が参照。
- **構造保持**: `docs/` はルートの構造をミラー。HTMLのタグ/属性/class/id/CSS/JSロジックは英語版と同一に保ち、テキストノードと指定属性のみ差し替える。相対リンクは変更しない（`docs/`内で完結）。
- **着地**: 追従は `main` へ直接 commit & push（PRなし）。GitHub Actions は使わない。
- **スケジュール**: 週1、毎週月曜 09:00 JST（= 00:00 UTC）。

- [ ] **Step 3: `Workflow` で並列翻訳を実行**

`Workflow` ツールを以下のスクリプトで起動する。`args` は Step 1 で作った `[{src, dst}, ...]` の配列（33要素）。`GLOSSARY` プレースホルダは `.i18n/glossary.md` の全文で置換する。各エージェントは `agentType: 'general-purpose'`（Read/Write 可能）。

```javascript
export const meta = {
  name: 'ja-i18n-initial-translation',
  description: 'Translate 33 English sample HTML files into Japanese under docs/',
  phases: [{ title: 'Translate', detail: 'one agent per HTML file' }],
}

const POLICY = `あなたは英語の自己完結HTMLを日本語に翻訳します。以下を厳守してください。

翻訳する: 画面表示テキスト、<title>、<meta name="description"> の content、alt/aria-label 属性。<html lang="en"> は <html lang="ja"> に変更。
訳さない（原文維持）: <code>/<pre>/<script>/<style> の中身、コード識別子・変数/関数/ファイル名、技術用語の英語表記、架空ブランド「Acme」等の固有名、先頭の <!-- Copyright ... --> ヘッダ。
文体: 敬体（です・ます）。UIラベル・ボタン・短い見出しは体言止め可。
構造保持: タグ/属性/class/id/CSS/JSロジックは英語版と完全に同一。テキストノードと指定属性のみ差し替える。ファイルの追加・削除・リンクの書き換えはしない。
訳語は下記の用語集に従う。`

const GLOSSARY = `__GLOSSARY_FULL_TEXT__`

phase('Translate')
const results = await parallel(args.map((f) => () =>
  agent(
    `${POLICY}\n\n# 用語集\n${GLOSSARY}\n\n# タスク\n入力ファイル \`${f.src}\` を Read し、上記ルールで日本語に翻訳した全文を \`${f.dst}\` に Write してください（親ディレクトリが無ければ作成）。完了したら "DONE ${f.dst}" とだけ返してください。`,
    { label: `translate:${f.dst}`, phase: 'Translate', agentType: 'general-purpose' }
  )
))
return { written: results.filter(Boolean).length, total: args.length }
```

Expected: `written === total === 33`。Workflow完了通知を待つ。

```markdown
# 週1 upstream追従・翻訳ルーティン指示書

あなたは html-effectiveness Fork の日本語i18n追従を自動実行するエージェントです。
以下を順に実行してください。**不変条件を厳守**すること。

## 不変条件（違反禁止）
- upstream追跡ファイル（ルートの `*.html`, `README.md`, `LICENSE` 等）は絶対に編集しない。
- 追加・変更してよいのは `docs/` と `.i18n/` の新規/既存パスのみ。
- 翻訳ポリシーと用語集（`.i18n/glossary.md`）に必ず従う。

## 手順
1. `git fetch upstream` を実行。
2. `git merge upstream/main` を `main` に対して実行。
   - **クリーンにマージできない場合は直ちに中止**し、pushせず、状況を通知して終了（異常事態）。
3. `.i18n/sync-state.json` の `lastUpstreamSha` を読む。これを `<PREV>`、現在の `upstream/main` のSHAを `<NOW>` とする。
4. `git diff --name-status <PREV> <NOW> -- '*.html' ':(exclude)docs/*'` で、ルート英語HTMLの変更(M)/新規(A)/削除(D)を検出。
5. 差分がゼロなら、何もコミットせず終了（no-op）。
6. 変更(M)・新規(A)の各ファイルを、下記ポリシー＋用語集で翻訳し、`docs/` の対応パス（先頭に `docs/` を付与）へ Write。
7. 削除(D)の各ファイルは、`docs/` の対応パスを削除。
8. `.i18n/sync-state.json` を `{ "lastUpstreamSha": "<NOW>", "lastSyncedAt": "<現在時刻ISO8601>" }` に更新。
9. 変更を `git add docs .i18n/sync-state.json` してコミットし、`main` へ**直接 push**（PRは作らない）。
   - コミットメッセージ例: `chore(i18n): sync translations with upstream <NOW短縮SHA>`
10. 実行結果（翻訳した/削除したファイル数、または no-op、または異常中止）を通知。

## 翻訳ポリシー
- 翻訳する: 画面表示テキスト、`<title>`、`<meta name="description">` の content、`alt`/`aria-label`、`<html lang="en">`→`<html lang="ja">`。
- 訳さない: `<code>`/`<pre>`/`<script>`/`<style>` 内、コード識別子・変数/関数/ファイル名、技術英語表記、架空ブランド「Acme」等の固有名、先頭の `<!-- Copyright ... -->`。
```
