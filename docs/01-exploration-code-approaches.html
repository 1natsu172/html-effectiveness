デバウンス検索を実装する3つの方法

プロンプト 私たちのReactコードベースで、タスクフィルター入力欄のデバウンス検索を実装する3つの異なる方法を、それぞれのトレードオフとともに見せてください。

01インラインの useEffect + setTimeout

デバウンスのロジックを、入力欄を持つコンポーネントの内部に直接書きます。

export function TaskSearch() {
  const [draft, setDraft] = useState('');
  const [query, setQuery] = useState('');

  useEffect(() => {
    const id = setTimeout(() => setQuery(draft), 300);
    return () => clearTimeout(id);
  }, [draft]);

  const { data } = useTasks({ search: query });

  return (
    <input
      value={draft}
      onChange={(e) => setDraft(e.target.value)}
      placeholder="Filter tasks…"
    />
  );
}

02カスタムの useDebounce フック

タイマーを src/hooks/ 配下の共有フックに切り出します。

// src/hooks/useDebounce.ts
export function useDebounce<T>(value: T, ms = 300): T {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), ms);
    return () => clearTimeout(id);
  }, [value, ms]);
  return debounced;
}

// TaskSearch.tsx
const [draft, setDraft] = useState('');
const query = useDebounce(draft, 300);
const { data } = useTasks({ search: query });

03小さな外部ライブラリ

値とコールバックの両方に use-debounce を採用します。

import { useDebouncedCallback }
  from 'use-debounce';

export function TaskSearch() {
  const [query, setQuery] = useState('');

  const onChange = useDebouncedCallback(
    (next: string) => setQuery(next),
    300,
    { leading: false, maxWait: 1000 },
  );

  const { data } = useTasks({ search: query });

  return (
    <input
      defaultValue=""
      onChange={(e) => onChange(e.target.value)}
    />
  );
}
メリット
デメリット
leading / trailing / maxWait を代わりに処理してくれる
監査と更新が必要なランタイム依存が増える
コールバック形式なら途中の再レンダリングを飛ばせる
非制御の入力欄がAcmeのフォームパターンから外れる
エッジケースが十分にテスト済み（unmount、flush、cancel）
自前で持てるもののために gzip 後 約1.4 kb
バンドルへの影響: +1.4 kb テスト容易性: 高 再利用性: 高 SSR安全: はい

推奨

アプローチ02、カスタムの useDebounce フックを選びます。Acmeにはインラインパターンを手書きしている箇所がすでに3つあり（タスクフィルター、コマンドパレット、メンバーピッカー）、共有フックを1つ切り出せば、新しい依存を抱え込まずに重複を解消できます。

後から maxWait や flush() の挙動が必要になった場合にのみ、アプローチ03を再検討します。要件が10行のフックで収まらなくなれば、ライブラリはそのバンドルコストに見合う価値を持ちます。
