04

主要コード

最も間違えやすい2つの部分は、マイグレーション（ソフトデリート、既読カーソル）と、楽観的ミューテーション（一時idの解決）です。

packages/db/migrations/0042_comments.sql
create table comments (
  id          uuid primary key default gen_random_uuid(),
  task_id     uuid not null references tasks(id),
  parent_id   uuid references comments(id),   -- one level only,
                                                -- enforced in API
  author_id   uuid not null references users(id),
  body        text not null,
  created_at  timestamptz not null default now(),
  deleted_at  timestamptz                       -- soft delete
);

create table comment_reads (
  task_id    uuid not null references tasks(id),
  user_id    uuid not null references users(id),
  read_up_to timestamptz not null,
  primary key (task_id, user_id)
);

create index comments_task_created
  on comments (task_id, created_at);
apps/web/hooks/useAddComment.ts
export function useAddComment(taskId: string) {
  const qc = useQueryClient();
  return trpc.comments.create.useMutation({
    onMutate: async (input) => {
      const temp = { ...input, id: `temp-${nanoid()}`,
                     createdAt: new Date(), pending: true };
      qc.setQueryData(key(taskId), (prev) =>
        [...(prev ?? []), temp]);
      return { tempId: temp.id };
    },
    onSuccess: (row, _v, ctx) => {
      // reconcile temp id → real id so the
      // realtime append doesn't duplicate it
      qc.setQueryData(key(taskId), (prev) =>
        prev.map((c) => c.id === ctx.tempId ? row : c));
    },
    onError: (_e, _v, ctx) => {
      qc.setQueryData(key(taskId), (prev) =>
        prev.filter((c) => c.id !== ctx.tempId));
    },
  });
}

リスクと対策

リスク
深刻度
対策
リアルタイムの重複: ソケットによる追加が、HTTPレスポンスや一時idの解決と競合します。
HIGH
キャッシュ更新側で、サーバーが割り当てたidにより重複を排除します。ソケットのペイロードは実idを運び、一時行は解決時にフィルタされます。
ユーザーが別のデバイスでスレッドを読むと、未読件数が古くなります。
MED
同じチャンネルでcomment_readsのupsertをブロードキャストします。クライアントは自身のカーソルをmax(local, remote)として扱います。
貼り付けたmarkdownで、メンション検出が誤検知します（@media、@2x）。
LOW
メンションはワークスペースのメンバーに対してのみ、書き込み時に解決し、解決済みのユーザーidを保存します。読み込み時に再パースはしません。

未解決の課題

編集を許可するか、それとも削除して再投稿のみにするか。
編集にはedited_atカラムと「編集済み」の表示が必要です。削除して再投稿する方式はシンプルですが、返信のアンカーが失われます。v1では削除のみに寄せる方向です。
